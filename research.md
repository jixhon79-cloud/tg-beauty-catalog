# Research: Telegram Mini App — Каталог услуг бьюти-мастера

> Дата исследования: 11 апреля 2026  
> Источник: анализ конкурентов, UX-паттерны, экспертная база (данные до августа 2025)

---

## 1. Обзор рынка: 5 примеров Telegram Mini Apps в нише

---

### 1.1 YCLIENTS — виджет онлайн-записи в Telegram

**Что делает:** крупнейшая CRM для салонов красоты в СНГ. Виджет записи открывается внутри Telegram как Mini App — выбор услуги, мастера, времени без перехода на сторонний сайт.

**Сильные стороны:**
- Тысячи подключённых заведений, зрелая платформа
- Синхронизация расписания в реальном времени
- Онлайн-оплата (Тинькофф, ЮKassa)
- Напоминания через Telegram-бота

**Слабые стороны:**
- Нет витрины мастера: портфолио, фото работ, рейтинги отсутствуют
- Интерфейс ощущается как "виджет сайта", не нативное Mini App
- Высокий порог входа (~2000–5000 руб/мес) — частные мастера не используют
- Нет возможности "открыть" нового мастера — только запись к известному заведению

**Что мы можем сделать лучше:** дать каждому мастеру красивую личную витрину + бесплатный порог входа

---

### 1.2 Dikidi — онлайн-запись для частных мастеров

**Что делает:** аналог YCLIENTS, ориентированный на частных мастеров и мелкие студии. Мастер получает личную ссылку-Mini App для клиентов с расписанием, услугами, ценами.

**Сильные стороны:**
- Ориентирован именно на частных мастеров
- Бесплатный базовый тариф — низкий порог входа
- Есть базовая галерея фото работ
- Личная ссылка мастера — удобно для сарафанного радио

**Слабые стороны:**
- Нет каталога-маркетплейса: нельзя найти нового мастера через приложение
- Дизайн устаревший, тёмная тема работает нестабильно
- Нет системы отзывов и рейтингов внутри Mini App
- Уведомления только при отдельной подписке на бота

**Что мы можем сделать лучше:** нативный дизайн под Telegram + верифицированные отзывы

---

### 1.3 Taplink в Telegram — витрина мастера

**Что делает:** сервис "ссылки в bio", поддерживает формат Telegram Mini App. Бьюти-мастера используют как витрину: портфолио, прайс, кнопка связи, ссылки на соцсети.

**Сильные стороны:**
- Визуально привлекательные шаблоны под бьюти-эстетику
- Настройка за 15 минут без кода
- Блоки: фото/видео портфолио, прайс-лист, отзывы, форма заявки
- Широко распространён — клиенты уже знакомы с форматом

**Слабые стороны:**
- Запись на услугу не встроена — только редирект на WhatsApp или телефон
- Нет управления расписанием — это витрина, не система записи
- Полный функционал только на платных тарифах
- Нет агрегации мастеров: нельзя найти кого-то нового

**Что мы можем сделать лучше:** объединить красивую витрину + реальную систему записи в одном продукте

---

### 1.4 Кастомные Mini Apps под бренд мастера

**Что делает:** часть активных бьюти-мастеров заказывает индивидуальные Telegram Mini Apps у фрилансеров. Стандартный набор: каталог услуг с фото, выбор мастера, слот-пикер, подтверждение через бота.

**Сильные стороны:**
- Нативный дизайн под конкретный бренд
- Полная интеграция с Telegram (тема, цвета, haptic feedback)
- Мгновенные уведомления через бота
- Нет чужого брендинга сторонних платформ

**Слабые стороны:**
- Высокая стоимость разработки (50–200 тыс. руб. за MVP)
- Нет масштабируемости — каждый мастер заказывает с нуля
- Зависимость от конкретного разработчика
- Редко есть онлайн-оплата, аналитика, авто-напоминания

**Что мы можем сделать лучше:** дать качество кастомного решения по цене SaaS-сервиса

---

### 1.5 Агрегаторы услуг (Profi.ru и аналоги)

**Что делает:** агрегаторы типа Profi.ru экспериментируют с Mini App: пользователь ищет мастера рядом, видит рейтинги, портфолио, цены, записывается прямо в Telegram.

**Сильные стороны:**
- Настоящий каталог с поиском и фильтрацией
- Рейтинги, отзывы, верификация мастеров
- Геолокация для поиска "рядом"
- Большая база исполнителей

**Слабые стороны:**
- Mini App — лишь дополнение к сайту, UX "срезан"
- Высокая комиссия снижает мотивацию мастеров
- Интерфейс перегружен (наследует логику десктопа)
- Слабая Telegram-нативность: нет haptic feedback, нет тёмной темы, нет MainButton

**Что мы можем сделать лучше:** нативный Telegram-опыт с честной монетизацией

## 3. Обязательные UI-элементы

### Навигация
- **Bottom Navigation Bar** (4–5 пунктов: Главная, Каталог, Запись, Мои записи)
- **Tab Bar с категориями** поверх списка услуг (горизонтальный скролл)
- **BackButton** из Telegram SDK — не своя кнопка «назад»

### Карточки услуг
- **Горизонтальная карточка**: фото 80×80px, название + описание + цена справа
- **Вертикальная сетка (2 колонки)** для визуальных услуг (маникюр, окрашивание)
- **Чип с длительностью** («45 мин») — обязателен
- **Бейджи** «Популярное» / «Новинка» / «Акция»

### Слоты и календарь
- **Инлайн-календарь** (не модальный попап) — выше конверсия
- **Слоты** в виде чипов: зелёные — свободны, серые — заняты
- **Горизонтальный скролл дат** — лёгкая альтернатива полному календарю

### Прочие стандарты
- **Sticky MainButton** внизу (через `Telegram.WebApp.MainButton`)
- **Progress Indicator** шагов записи («Шаг 2 из 3»)
- **Skeleton Loader** вместо спиннера
- **Toast/Snackbar** для быстрых уведомлений
- **HapticFeedback** при подтверждении

## 4. Функции, ожидаемые пользователем

### Критически важные
1. **Автозаполнение** имени и контакта из Telegram-профиля
2. **Видимость занятых слотов** в реальном времени
3. **Мгновенное подтверждение** записи (или явный статус «ожидает»)
4. **Напоминания в Telegram** за 24 ч и за 2 ч до визита (с кнопками прямо в сообщении)
5. **Отмена/перенос** без звонка мастеру — прямо в приложении
6. **Итоговая стоимость** без скрытых сюрпризов до нажатия «Подтвердить»

### Важные
7. **История посещений** — «в прошлый раз была на маникюре 3 недели назад»
8. **Повторная запись** одним кликом из истории
9. **Выбор нескольких услуг** в одно посещение
10. **Адрес + кнопка** «Открыть в картах»
11. **Фото работ** по конкретной услуге — «хочу так же»
12. **Статус записи**: подтверждена / ожидает / отменена

## 5. Типичные ошибки при разработке

### UX/Дизайн
| Ошибка | Решение |
|--------|---------|
| Флоу записи 5+ шагов | Максимум 3 шага: услуга → дата/время → подтверждение |
| Модальный попап для календаря | Инлайн-календарь прямо на экране |
| Нет индикатора прогресса | «Шаг 2 из 3» — обязателен |
| Мелкие цели нажатия (<44px) | Минимум 44×44px для всех интерактивных элементов |
| Нет пустых состояний | Иллюстрация + текст + CTA при отсутствии данных |

### Специфично для Telegram Mini App
| Ошибка | Решение |
|--------|---------|
| Не используется `MainButton` | Использовать `Telegram.WebApp.MainButton` |
| Фиксированные цвета | CSS-переменные `var(--tg-theme-*)` для авто-адаптации к теме |
| Нет HapticFeedback | Вибрация при подтверждении через Telegram SDK |
| BackButton не обработан | При нажатии — переход назад, не закрытие |
| Тяжёлые изображения | Ленивая загрузка, WebP, оптимизация |

## 6. Что сделать лучше конкурентов: 5 ключевых улучшений

### #1 Нулевое трение при записи
- Telegram ID = авторизация. Никаких форм регистрации.
- 3 шага максимум: Услуга → Дата/Время → Подтверждение
- «Повторная запись» — 1 tap

### #2 Живое портфолио как основа доверия
- Галерея реальных работ с тегами услуг прямо в карточке
- Верифицированные отзывы только от клиентов с состоявшейся записью (автозапрос через бота)
- «Хочу так же» — сохранить работу как референс для мастера

### #3 Умные напоминания внутри Telegram
- Напоминания через бота с кнопками «Подтвердить» / «Перенести» / «Отменить» прямо в сообщении
- Открываемость Telegram-сообщений ~80–90% против 5–10% у push-уведомлений
- Мастер получает уведомление о новой записи мгновенно в Telegram

### #4 Прозрачное ценообразование
- Чёткая цена или честный диапазон с объяснением, что влияет на итог
- Политика отмены — крупно, до нажатия «Записаться»
- Никаких «от» без расшифровки

### #5 История и персонализация
- «Ваше последнее окрашивание было 7 недель назад — пора на коррекцию»
- Быстрое повторное бронирование у того же мастера
- Накопительная система лояльности прямо в Mini App

## 7. Уникальные фичи для WOW-эффекта

| Фича | Описание |
|------|---------|
| **Waitlist** | Лист ожидания — при освобождении слота бот уведомляет с кнопкой «Занять» |
| **Видео-визитка мастера** | 15–30 сек видео в профиле — эмоциональный контакт до первого визита |
| **Спросить мастера** | Анонимный вопрос через бота до записи (снижает барьер входа) |
| **«Горящие» слоты** | Скидка 10–15% на свободное окно через 2–3 часа → автоматический оффер подписчикам |
| **Групповая запись** | Записаться вместе с подругой на одно время |
| **Sharable-карточки** | Кнопка в канале → Mini App сразу на нужной услуге |
| **Telegram Payments** | Оплата депозита без редиректа на внешний сайт |

## 8. Визуальный стиль для бьюти-ниши

- **Палитра:** пастельная или нейтральная — розовый, пудровый, беж, молочный, лавандовый
- **Скругления:** 16–24px для карточек и кнопок
- **Типографика:** Inter, Montserrat, Nunito — тонкая санс-сериф или элегантный сериф для заголовков
- **Одна акцентная кнопка** — не распылять цветовые акценты
- **Качественные фото работ** — это №1 в бьюти, без них всё остальное теряет смысл
- **Анимация успеха** — конфетти или анимированная галочка при подтверждении записи
- **Плавные переходы** между экранами (slide, fade)
- **Описания услуг на языке результата** — «Стойкое окрашивание до 8 недель» вместо «Окрашивание волос»

## 10. Экспертная оценка

---

### 10.1 Взгляд UX-дизайнера

#### Ошибки новичков (которые убивают конверсию)

**1. Слишком длинный флоу записи.**
Новички проектируют "как в голове" — услуга → категория → подуслуга → мастер → дата → время → ваши данные → подтверждение. 8 шагов. Пользователь уходит на 3–4-м. Правило: **максимум 3 экрана от входа до "Запись подтверждена"**.

**2. Обязательная регистрация.**
Форма "введите email и придумайте пароль" в Telegram Mini App — это катастрофа. Пользователь пришёл из мессенджера, у него уже есть аккаунт. Telegram ID и `initData` — единственная нужная "авторизация".

**3. Пустые состояния без дизайна.**
Новый мастер только что настроил приложение — у него нет отзывов, нет записей в истории, нет акций. Если показать пустой белый экран вместо "Отзывов пока нет — станьте первым!", это ощущение брошенного продукта. Каждый пустой экран требует иллюстрации + текста + призыва к действию.

**4. Каталог без группировки.**
Свалка из 20 услуг одним списком без категорий — пользователь теряется. Категории с горизонтальным скроллом (таб-бар) обязательны уже с первой версии.

**5. Нет состояний загрузки.**
Skeleton-лоадеры вместо спиннера создают ощущение быстрого приложения. Спиннер на 2 секунды = ощущение лагов.

#### Что обязательно должно быть

- **Один главный CTA на экране** — глаз не должен гадать, куда нажать. На главной: одна кнопка «Записаться».
- **Цена видна до входа в карточку** — пользователь сканирует список и должен видеть цену в карточке каталога, не только внутри.
- **Длительность услуги** — «45 мин» рядом с ценой. Это влияет на решение не меньше цены.
- **Фото работ в карточке услуги** — минимум 2–3 реальных фото. Без них карточка не работает в бьюти.
- **Экран подтверждения (Summary)** перед финальным нажатием — дать пользователю секунду убедиться, что всё правильно.

#### Что бесит пользователей (по данным исследования конкурентов)

- Занятые слоты не показаны — выбираешь время, нажимаешь, получаешь "это время недоступно". Все занятые слоты должны быть серыми **до** нажатия.
- Нет возможности отменить запись без звонка — особенно критично, если мастер не берёт трубку.
- Цена "от" без объяснений — пришла на «от 1500», заплатила 3000. Потеря доверия навсегда.
- Нет подтверждения после записи — не ясно, прошло бронирование или нет.
- Форма, которая не запомнила имя — каждый раз вводить заново.

### 10.2 Взгляд маркетолога

#### Что заставит записаться (психология конверсии)

**Социальное доказательство — главный триггер.**
Бьюти — это доверие. До записи пользователь должен увидеть реальные работы и реальные отзывы. Формула: *фото работы → отзыв с именем и фото клиента → кнопка «Записаться»*. Разместить этот блок на главном экране ещё до каталога.

**Дефицит и срочность.**
«Осталось 2 свободных места на этой неделе» — работает. «Ближайшая запись: завтра в 14:00» рядом с кнопкой «Записаться» резко поднимает кликабельность. Показывать ближайший доступный слот без лишних кликов.

**Снижение барьера первого визита.**
Для новых клиентов: блок «Первый раз у меня? → Задать вопрос мастеру» или «-10% на первую запись». Люди боятся ошибиться с выбором мастера — дать им способ убедиться до записи.

**Описания услуг на языке результата.**
«Маникюр с покрытием» не продаёт. «Стойкий маникюр до 3 недель — не скалывается, не тускнеет» продаёт. Переписать все описания под формулу: *что получишь + как долго держится + из чего состоит*.

**Лояльность через прогресс.**
Полоска прогресса «3 из 5 визитов до подарка» в профиле — человек видит, что он уже близко, и это мотивирует записаться ещё раз. Работает значительно лучше, чем абстрактные «накопительные баллы».

#### Элементы, повышающие конверсию

| Элемент | Эффект |
|---------|--------|
| Фото мастера + имя на главной | Личный контакт → доверие |
| Реальные фото работ в карточке | Покупают глазами — особенно в бьюти |
| Ближайший свободный слот рядом с кнопкой | Снимает усилие «а когда можно?» |
| Счётчик клиентов «248 человек уже записались» | Социальное доказательство |
| Бейдж «Популярное» на топ-услугах | Подсказка для нерешительных |
| Автозаполнение из Telegram | Убирает трение, ускоряет финальный шаг |
| Кнопка «Повторить запись» в истории | Возвращающийся клиент записывается в 1 тап |
| Промокод при первой записи через бота | Снижает барьер для нового клиента |

#### Ошибки маркетинга, которые убивают продажи

- **Нет чёткого позиционирования на главной** — кто этот мастер, что он делает лучше всего, для кого. Первые 3 секунды решают, останется пользователь или нет.
- **Все услуги равнозначны** — нет выделения топ-услуг и хитов. Мастер должен вести клиента к нужному выбору.
- **Нет offboarding-цикла** — после успешного визита нет автоматического напоминания «Пора обновить стрижку» через 4 недели. Это самый дешёвый способ вернуть клиента.

### 10.3 Взгляд Telegram-разработчика

#### Ограничения платформы (важно знать заранее)

**WebView — не нативное приложение.**
Mini App работает в встроенном браузере Telegram. Это значит:
- Нет доступа к файловой системе (кроме `<input type="file">`)
- CSS-анимации должны быть лёгкими — тяжёлые анимации лагают на слабых устройствах
- Нет фонового выполнения кода — приложение "засыпает" когда свёрнуто
- WebGL и Canvas работают, но с ограничениями на iOS (WKWebView)

**Размер приложения.**
Первый экран должен открываться быстро. Telegram не кэширует Mini App агрессивно. Рекомендации:
- Бандл JS < 200 КБ (gzip)
- Критические стили inline или в первом чанке
- Ленивая загрузка всего, что не нужно на первом экране
- Изображения в WebP, загрузка через `loading="lazy"`

**`initData` и безопасность.**
Telegram передаёт данные пользователя через `window.Telegram.WebApp.initData`. Эти данные **обязательно нужно валидировать на бэкенде** через HMAC-SHA256 с ботовым токеном. Без валидации любой может подделать запрос и записаться от имени другого пользователя.

**Высота экрана непредсказуема.**
На разных устройствах и версиях Telegram высота Mini App разная. Использовать `window.Telegram.WebApp.viewportHeight` и `viewportStableHeight` вместо `100vh`. Иначе контент будет обрезан клавиатурой или системными элементами.

**iOS vs Android различия.**
- HapticFeedback работает только на мобильных (не на десктопном Telegram)
- `MainButton` на iOS иногда перекрывает нижний контент — добавлять `padding-bottom` динамически
- Скролл внутри Mini App на iOS требует `-webkit-overflow-scrolling: touch` или использование `overscroll-behavior`

#### Что работает хорошо

**`MainButton` — главная кнопка платформы.**
`Telegram.WebApp.MainButton` — нативная кнопка в нижней части экрана. Она физически за пределами WebView, рендерится самим Telegram. Это значит: всегда видна, не перекрывается контентом, работает быстрее DOM-кнопки. Использовать для главного CTA на каждом экране.

**`BackButton` — бесплатная навигация.**
`Telegram.WebApp.BackButton` — нативная кнопка «назад» в хедере Telegram. Показывать/скрывать программно в зависимости от экрана. Пользователи ожидают этого поведения — если не реализовать, они нажмут и Mini App закроется.

**Тема Telegram — автоматическая адаптация.**
CSS-переменные `--tg-theme-bg-color`, `--tg-theme-text-color`, `--tg-theme-button-color` и другие автоматически меняются при переключении темы. Строить весь дизайн на этих переменных = приложение выглядит нативно в любой теме без дополнительного кода.

**`sendData` vs `openLink` vs бот-сообщения.**
- `sendData` — отправить данные боту при закрытии Mini App (для простых сценариев)
- `openTelegramLink` — открыть другой Telegram-чат или канал без выхода из приложения
- Для записи: лучше API-запрос к бэкенду → бот отправляет подтверждение через Bot API. Это надёжнее и позволяет отправлять структурированные сообщения с кнопками.

**CloudStorage — хранилище на стороне Telegram.**
`Telegram.WebApp.CloudStorage` — до 1024 значений (key-value), хранится на серверах Telegram, доступно на всех устройствах пользователя. Использовать для: сохранения выбранных фильтров, последней услуги, состояния онбординга. Не хранить там записи — только UI-состояние.

#### Что лучше не делать

| Анти-паттерн | Почему плохо | Альтернатива |
|---|---|---|
| `window.location.href` для навигации | Перезагружает Mini App, теряет состояние | React Router / Vue Router с history API |
| `alert()` / `confirm()` | Не работают в Telegram WebView | `Telegram.WebApp.showAlert()` / `showConfirm()` |
| Фиксированные hex-цвета | Ломается в тёмной теме | `var(--tg-theme-*)` везде |
| `localStorage` для пользовательских данных | Не синхронизируется между устройствами | CloudStorage или бэкенд |
| Тяжёлые CSS-анимации (3D, blur) | Лаги на слабых Android-устройствах | `transform` + `opacity` — самые дешёвые свойства |
| Открывать внешние ссылки через `window.open` | Может быть заблокировано | `Telegram.WebApp.openLink()` |
| Несколько `MainButton` состояний без `disable` | Двойные нажатия = двойная запись | Сразу после нажатия: `MainButton.disable()` |
| Запросы к API без debounce | Шквал запросов при быстром скролле | Debounce 300 мс на все поисковые запросы |

#### Рекомендованный стек для MVP

```
Фронтенд:   React + Vite (быстрая сборка, маленький бандл)
Стили:      CSS Modules или Tailwind (не CSS-in-JS — медленнее)
Telegram SDK: @telegram-apps/sdk или прямой window.Telegram.WebApp
Бэкенд:     Node.js (Fastify) или Python (FastAPI) — что ближе
БД:         PostgreSQL (записи, услуги) + Redis (слоты, кэш)
Бот:        grammy (JS) или aiogram (Python)
Деплой:     Railway / Render / VPS — нужен HTTPS, Mini App работает только по HTTPS
```

### 10.4 Общий список рекомендаций

#### Критически важно (сделать до запуска)
1. **Флоу записи — максимум 3 шага.** Любое усложнение режет конверсию.
2. **Валидировать `initData` на бэкенде** — безопасность, не компромисс.
3. **Все слоты отображать как занятые/свободные до нажатия** — никаких сюрпризов.
4. **Использовать `MainButton` и `BackButton`** — не делать свои аналоги.
5. **CSS через `var(--tg-theme-*)` везде** — тёмная тема обязательна.
6. **Высота через `viewportStableHeight`**, не `100vh`.
7. **Мгновенное подтверждение записи** через бота — сразу после нажатия.
8. **Skeleton-лоадеры** вместо спиннеров.
9. **Пустые состояния** на каждом экране — иллюстрация + текст + CTA.
10. **Отмена записи** внутри приложения — без звонка мастеру.

#### Важно для конверсии (влияет на продажи)
11. Фото реальных работ в карточках услуг — минимум 2–3 штуки.
12. Цена и длительность видны в списке каталога, не только в карточке.
13. Ближайший доступный слот показывать рядом с кнопкой «Записаться».
14. Описания услуг на языке результата, не технические названия.
15. Социальное доказательство на главном экране: фото работ + отзывы.
16. Верифицированные отзывы (только от клиентов с состоявшейся записью).
17. Промокод или скидка для новых клиентов — снижает барьер первого визита.
18. После успешного визита — автоматический запрос отзыва через бота (через 24 ч).

#### Хорошо для удержания (влияет на LTV)
19. История посещений с умными напоминаниями («пора на коррекцию»).
20. Повторная запись в 1 tap из истории.
21. Прогресс лояльности в профиле («3 из 5 визитов до подарка»).
22. Waitlist с авто-уведомлением при освобождении слота.
23. «Горящие» слоты со скидкой — конвертирует мёртвые часы.

#### Технические детали (для качества)
24. `MainButton.disable()` сразу после нажатия — защита от двойной записи.
25. `CloudStorage` для UI-состояния, не `localStorage`.
26. `Telegram.WebApp.showAlert()` вместо `window.alert()`.
27. Бандл JS < 200 КБ, изображения в WebP.
28. HTTPS обязателен — без него Mini App не откроется.
29. Debounce на поисковые/фильтрующие запросы.
30. `HapticFeedback` при подтверждении записи и при ошибке.
