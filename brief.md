# Brief: Telegram Mini App — Каталог услуг бьюти-мастера

> Версия: MVP (v1)  
> Дата: 11 апреля 2026  
> Источник: research.md

---

## Концепция одной фразой

**Личная витрина бьюти-мастера внутри Telegram: смотришь работы → выбираешь услугу → записываешься за 3 шага, не выходя из мессенджера.**

---

## Принципы дизайна

- Нативный Telegram-стиль: цвета из `var(--tg-theme-*)`, адаптация к светлой/тёмной теме автоматически
- Все кнопки минимум **44×44px** — попадание пальцем без промахов
- Минимум текста, максимум визуала — фото работ, иконки, чипы
- Максимум **3 шага** от входа до «Запись подтверждена»
- Один главный CTA на каждом экране
- `MainButton` (нативная кнопка Telegram) для всех основных действий
- `BackButton` (нативная кнопка Telegram) для навигации назад
- Русский язык интерфейса

---

## Навигация (Bottom Tab Bar)

```
[ Главная ]  [ Услуги ]  [ Записи ]
```

3 вкладки. Минималистично. Запись — не отдельная вкладка, а флоу из карточки услуги.

**Что видит пользователь:**

```
┌─────────────────────────────┐
│  [Фото мастера — круглое]   │
│  Имя мастера                │
│  Специализация / tagline    │
│                             │
│  ┌─────────────────────┐    │
│  │  ⭐ 4.9  · 128 работ │    │
│  └─────────────────────┘    │
│                             │
│  — — — Мои работы — — —     │
│  [фото] [фото] [фото] →     │
│                             │
│  — — — Отзывы — — —         │
│  ┌───────────────────────┐  │
│  │ 👤 Анна · «Очень...»  │  │
│  └───────────────────────┘  │
│                             │
│  📍 Москва, Арбат · Карта   │
└─────────────────────────────┘
       [MainButton: Записаться]
```

**Элементы:**
- Аватар мастера (круглое фото, 80×80px)
- Имя и специализация (напр. «Мастер маникюра»)
- Рейтинг + количество работ — одна строка
- Горизонтальная галерея работ (3–4 фото, стрелка «ещё»)
- Один отзыв-превью (имя + первые слова)
- Адрес + кнопка «Открыть карту» (иконка 📍)

**Действия:**
- Нажать на фото в галерее → открывается Экран 6 (Галерея)
- Нажать на отзыв → открывается Экран 7 (Отзывы)
- Нажать «Открыть карту» → `Telegram.WebApp.openLink()` в Яндекс.Карты
- `MainButton «Записаться»` → переход на Экран 2 (Каталог услуг)

### Экран 4 — Выбор даты и времени

**Что видит пользователь:**

```
┌─────────────────────────────┐
│  ← Шаг 1 из 2              │
│                             │
│  Выберите дату              │
│                             │
│  Апр 2026                   │
│  Пн Вт Ср Чт Пт Сб Вс      │
│  .. .. 1  2  3  [4] 5       │  ← [4] = сегодня, жирный
│  6  7  8  9  10 11 12       │
│  13 ✕  15 16 17 18 19       │  ← ✕ = выходной/занят
│  ...                        │
│                             │
│  Выберите время             │
│                             │
│  [09:00] [10:00] [11:00]    │  ← чипы, зелёные = свободны
│  [12:00✕] [13:00] [14:00]   │  ← серые = заняты
│  [15:00] [16:00] [17:00]    │
└─────────────────────────────┘
   [MainButton: Далее →] (активна после выбора слота)
```

**Элементы:**
- Индикатор шага: «Шаг 1 из 2»
- Инлайн-календарь (не модальный попап):
  - Рабочие дни — активны
  - Выходные/полностью занятые — серые, не нажимаются
  - Сегодня — выделен
  - Выбранная дата — акцентный цвет
- Слоты времени — чипы 2-3 в ряд:
  - Зелёные/акцентные — свободны, нажимаются
  - Серые, зачёркнутые — заняты, не нажимаются
- `MainButton «Далее»` — неактивна пока не выбраны дата + время

**Действия:**
- Нажать дату → обновить список слотов для этой даты
- Нажать слот → выделить, активировать MainButton
- `MainButton «Далее»` → Экран 5 (Подтверждение)
- `BackButton` → Экран 3

```
Главная

            └─ Карточка услуги
                 └─ [Записаться]

```

**Итого: 5 экранов, 4 нажатия. Без регистрации.**

## Чего НЕ будет в v1

| Функция | Причина отсутствия | Версия |
|---------|-------------------|--------|
| Онлайн-оплата / депозит | Требует Telegram Payments + юрлицо | v3 |
| Система отзывов (написать) | Нужна верификация «был на записи» | v2 |
| Портфолио как отдельная страница | Галерея на главной достаточна для MVP | v2 |
| О мастере (подробная биография) | Не влияет на первую запись | v2 |
| Waitlist (лист ожидания) | Сложная логика уведомлений | v3 |
| «Горящие» слоты со скидкой | Требует динамического ценообразования | v3 |
| Групповая запись | Редкий сценарий, сложная логика | v3 |
| Комбо-услуги (несколько за раз) | Сложный слот-пикер | v2 |
| Фильтры по цене / длительности | Излишне при небольшом каталоге | v2 |
| Поиск по услугам | Излишне при небольшом каталоге | v2 |
| Программа лояльности | Требует бэкенд-логики баллов | v3 |
| Геолокация «рядом» | Только один мастер, не агрегатор | v3 |
| Смена мастера | Один мастер на MVP | v2+ |
| Регистрация / профиль пользователя | Telegram ID = идентификатор | Никогда |

## Уведомления через бота (вне Mini App)

Бот отправляет сообщения автоматически:

| Событие | Когда | Содержание |
|---------|-------|-----------|
| Запись создана | Сразу | «Вы записаны: [услуга], [дата], [время]» |
| Напоминание 1 | За 24 ч | «Напоминаем о записи завтра в [время]» + кнопки [Подтвердить] [Отменить] |
| Напоминание 2 | За 2 ч | «Ждём вас сегодня в [время]» + адрес |
| Запись отменена | При отмене | «Запись отменена. Хотите записаться на другое время?» + кнопка |
| После визита | Через 24 ч | «Как прошёл визит? Оставьте отзыв» + кнопка (для v2) |

## Технические требования

- HTTPS обязателен
- Адаптация к теме: `var(--tg-theme-bg-color)`, `var(--tg-theme-text-color)`, `var(--tg-theme-button-color)` и др.
- Высота: `Telegram.WebApp.viewportStableHeight`, не `100vh`
- Авторизация: `initData` из Telegram SDK, валидация на бэкенде через HMAC-SHA256
- `MainButton.disable()` сразу после нажатия «Подтвердить» — защита от дублей
- `Telegram.WebApp.showConfirm()` для отмены записи (не `window.confirm`)
- `HapticFeedback.impactOccurred('medium')` при подтверждении записи
- `HapticFeedback.notificationOccurred('error')` при ошибке
- Изображения в WebP, ленивая загрузка
- Skeleton-лоадеры при загрузке данных
