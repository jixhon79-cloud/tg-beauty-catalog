# TESTING.md — Руководство тестировщика

## Что это за проект

Telegram Mini App: каталог услуг бьюти-мастера с онлайн-записью.  
Одностраничное приложение (SPA) на чистом HTML/CSS/JS, без фреймворков и сборщиков.

**Стек:** HTML5 · CSS3 · Vanilla JS · Telegram Web App SDK  
**Деплой:** Vercel  
**Нет:** серверной части, базы данных, авторизации — всё хранится локально в `state`.

---

## Быстрый запуск (локально)

### Требования
- Python 3 **или** Node.js — для локального HTTP-сервера
- Браузер (Chrome/Edge рекомендуется)

### Шаги
```bash
# 1. Перейти в папку проекта
cd tg-beauty-catalog

# 2. Запустить сервер — любой вариант:
python -m http.server 8080          # Python
# или
npx serve .                          # Node.js

# 3. Открыть в браузере
# http://localhost:8080
```

### Что работает в браузере, что — нет

| Функция | Браузер | Telegram |
|---------|---------|---------|
| Навигация между экранами | ✅ | ✅ |
| Каталог услуг, фильтры | ✅ | ✅ |
| Календарь, выбор времени | ✅ | ✅ |
| Флоу записи целиком | ✅ | ✅ |
| Кнопка MainButton (синяя кнопка внизу) | ❌ невидима | ✅ |
| Кнопка BackButton (стрелка вверху) | ❌ невидима | ✅ |
| HapticFeedback (вибрация) | ❌ | ✅ |
| Имя пользователя (из Telegram) | ❌ «Пользователь» | ✅ настоящее имя |
| Открыть карту (Яндекс) | ✅ в новой вкладке | ✅ |

> В браузере MainButton заменена на браузерную кнопку-бар сверху. В Telegram — нативная синяя кнопка Telegram снизу.

---

## Запуск в Telegram

1. Задеплоить на Vercel (или воспользоваться уже задеплоенной версией)
2. Создать бота через @BotFather → `/newbot`
3. В BotFather: Menu → Web App → указать HTTPS-ссылку на задеплоенный сайт
4. Открыть Mini App через бота

---

## Структура файлов

```
tg-beauty-catalog/
├── index.html          — точка входа, HTML-оболочка, таб-бар
├── css/style.css       — все стили
├── js/data.js          — данные: мастер, услуги, отзывы, слоты
├── js/app.js           — вся логика: состояние, навигация, рендер
├── .env.example        — шаблон переменных (BOT_TOKEN)
└── vercel.json         — конфиг деплоя
```

**Менять данные** (имя мастера, услуги, цены, отзывы) — только в `js/data.js`.  
**Менять логику/экраны** — в `js/app.js`.

## Тест-кейсы

**Шаги:**
1. Открыть `http://localhost:8080`
2. Дождаться загрузки (исчезновение спиннера)

**Шаги:**

2. Нажать вкладку «Записи»

### TC-04 — Каталог услуг и фильтры по категориям

**Шаги:**
1. Перейти на вкладку «Услуги»
2. Нажать категорию «Маникюр»
3. Нажать категорию «Педикюр»
4. Нажать категорию «Все»

**Ожидаемый результат:**
- «Маникюр» → в списке только услуги категории «Маникюр» (3 шт.: гель-лак, без покрытия, снятие)
- «Педикюр» → только педикюр-услуги (2 шт.)
- «Все» → все 7 услуг
- Услуги с `popular: true` имеют бейдж «Популярное»
- У каждой карточки: фото/заглушка, название, длительность, цена

**Шаги:**

**Ожидаемый результат:**

### TC-06 — Полный флоу записи (Happy Path)

**Шаги:**
1. Из карточки «Маникюр с гель-лаком» нажать «Записаться»
2. В календаре выбрать любую доступную дату (не выходной, не прошедшую)
3. Выбрать свободный слот времени (не серый)
4. Нажать «Далее»
5. На экране подтверждения нажать «Подтвердить запись»

**Ожидаемый результат на каждом шаге:**
- **Шаг 1:** Открылся экран «Выбор даты и времени», индикатор «Шаг 1 из 2»
- **Шаг 2:** Выбранная дата подсвечена акцентным цветом, под календарём появились слоты
- **Шаг 3:** Слот выделен, кнопка «Далее» стала активной
- **Шаг 4:** Открылся экран подтверждения со сводкой: услуга + дата + время + цена
- **Шаг 5:** Экран успеха «Вы записаны!» с галочкой, в «Мои записи» появилась новая запись

### TC-07 — Блокировки в выборе даты/времени

**Шаги:**
1. Начать флоу записи
2. Попробовать нажать на прошедшую дату
3. Попробовать нажать на воскресенье
4. Попробовать нажать на серый (занятый) слот времени

**Ожидаемый результат:**
- Прошедшие даты — неактивны, не нажимаются
- Воскресенья — неактивны

- Кнопка «Далее» неактивна пока не выбраны оба: дата + время

### TC-08 — Отмена записи

**Шаги:**
1. Перейти в «Мои записи»
2. Нажать «Отменить» на предстоящей записи
3. В диалоге подтвердить отмену

- Если нажать «Нет» в диалоге — ничего не меняется

## Известные ограничения (не баги)

| Поведение | Причина |
|-----------|---------|
| Записи не сохраняются после перезагрузки | Нет бэкенда, `state` хранится в памяти |
| Занятые слоты меняются у разных дат, но фиксированы для одной даты | Псевдослучайный алгоритм на хэше даты — это заглушка, в проде заменить на API |
| Имя пользователя «Пользователь» в браузере | `tg.initDataUnsafe.user` доступен только в Telegram |
| Фото с Unsplash могут не загружаться | Заглушки — в проде заменить на реальные фото мастера |
| BOT_TOKEN в `.env` не используется приложением | Переменная для будущего бэкенда уведомлений |

## Переменные окружения

Файл `.env` нужен только если подключать Telegram-бота (уведомления).  
Для запуска и тестирования самого Mini App он **не нужен**.

```bash
cp .env.example .env
# Вставить BOT_TOKEN из @BotFather
```
